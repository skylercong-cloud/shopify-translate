# Shopify.dev 中文翻译代理设计

日期：2026-06-11  
状态：设计已确认，尚未进入实现  
用途：个人使用的技术文档阅读、翻译与检索工具

## 1. 目标

构建一个部署在中国大陆私人云服务器上的个人文档工具，覆盖
`shopify.dev` 下公开的 Docs、API Reference 和 Changelog。

系统提供：

- 简洁的独立文档阅读界面。
- 中文与英文整页切换。
- AI 按需初译与后台逐步覆盖。
- 专业术语、代码标识符和代码块保持英文原文。
- 中文译文、英文原文和技术标识符的统一搜索。
- 个人人工校正、术语维护和模型配置。
- 每日原文变化检查，仅重译变化段落。
- 每日数据库备份，保留 14 天。

系统不是 Shopify 官方中文站，不开放注册，不提供公开内容发布或经营服务。

## 2. 覆盖范围

### 2.1 包含

- `shopify.dev` 公开 Docs。
- 公开 API Reference。
- 公开 Changelog。
- 页面标题、正文、列表、表格、提示块、图片引用和导航层级。
- 原页面代码示例、代码注释、链接和文件路径。
- 指向官方高度交互工具的入口。

### 2.2 不包含

- Shopify 登录区域。
- Partner Dashboard。
- Shopify 社区、论坛及其他外部网站。
- 需要用户账号或授权才能访问的内容。
- 对在线 API Explorer 等复杂交互工具的复刻。
- 本地运行 AI 大模型。

## 3. 已确认的产品规则

### 3.1 翻译规则

- 普通自然语言翻译为简体中文。
- 专业术语保持英文。
- API 名称、GraphQL type、field、mutation、Liquid object、filter、tag
  和代码标识符保持英文。
- URL、命令、参数、文件名、配置键和返回字段保持英文。
- 代码块及代码块内的英文注释完全保持官方原文。
- 不允许模型补充、删减或推测原文没有的信息。
- 无法可靠翻译时显示英文原文，不发布错误译文。

### 3.2 语言切换

- 默认显示中文。
- 可切换到完整英文原文。
- 中文和英文都在内部应用中展示，不跳转页面。
- 切换时保持 URL、目录状态和阅读位置。
- 代码块在两种语言模式下完全一致。
- 浏览器记住上次选择的语言。

### 3.3 翻译状态

页面或段落具有以下状态：

- `等待翻译`
- `AI 初译`
- `人工已校正`
- `原文已更新，待复核`
- `翻译失败，显示英文`

人工校正不是所有页面发布的必经步骤。人工修改优先于 AI 译文；对应英文原文变化后，
系统保留历史校正记录，但将当前段落标记为待复核。

## 4. 阅读界面

采用“专注阅读模式”：

- 顶部提供全局搜索、语言切换和翻译状态。
- 左侧导航默认收窄，需要时展开完整目录。
- 正文区域保持较宽的舒适阅读宽度。
- 页内目录默认弱化，需要时展开。
- 页面明确显示 AI 初译、人工已校正等状态。
- 提供官方原页面入口和最近同步时间。
- 不复刻 Shopify 的完整视觉和复杂交互。

管理功能与阅读界面分离，避免配置项干扰日常阅读。

## 5. 总体架构

采用单服务器一体化部署，内部保持模块边界：

```text
浏览器
  |
  | HTTPS + 单用户登录
  v
Caddy / Nginx
  |
  v
Web 应用
  |-- 阅读与中英文切换
  |-- 搜索
  |-- 个人管理后台
  |
  |-- 抓取/同步 Worker
  |-- 翻译 Worker
  |-- 搜索索引更新
  |
  v
PostgreSQL
  |-- 原文与译文
  |-- 内容指纹与版本
  |-- 术语库与 Prompt 版本
  |-- 人工校正记录
  |-- 数据库任务队列
  |-- 中英文搜索索引
```

外部依赖：

- `shopify.dev` 公开页面与 sitemap。
- DeepSeek API 作为主模型。
- 通义千问 API 作为备用模型。

建议实现为 TypeScript 模块化单体，通过 Docker Compose 管理 Web、Worker、
PostgreSQL 和反向代理。任务队列优先使用 PostgreSQL 支持的持久化队列，
第一版不额外引入 Redis。

## 6. 内容获取与规范化

### 6.1 页面发现

- 读取 Shopify 官方 sitemap 发现公开页面。
- 仅处理允许访问且属于约定范围的 URL。
- 遵守站点访问限制，并设置低并发、超时、重试和请求间隔。
- 页面 URL 经过规范化，避免同一内容重复入库。

### 6.2 内容获取

- 优先读取 Shopify 页面提供的纯文本版本。
- 需要恢复表格、图片、导航和提示块结构时，再读取 HTML。
- 不执行来源页面中的第三方脚本。
- 抓取失败时保留上次成功版本并提供官方原页链接。

### 6.3 内容分段

页面被解析为稳定的结构化块：

- 标题
- 段落
- 列表项
- 表格单元
- 提示块
- 代码块
- 图片及替代文本

每个块保存结构路径、顺序、英文内容指纹和内容类型。代码块不可进入翻译队列。

## 7. AI 翻译流水线

### 7.1 翻译步骤

1. 解析页面并选出自然语言块。
2. 保护代码、URL、数字、标识符和锁定术语。
3. 通过内容指纹查询人工校正和翻译记忆。
4. 未命中内容发送给 DeepSeek。
5. 对模型结果执行结构与内容校验。
6. 恢复受保护内容。
7. 校验通过后保存译文并更新搜索索引。

翻译请求携带：

- 页面标题与导航层级。
- 当前块及必要的相邻上下文。
- 当前 Prompt 版本。
- 当前术语库版本。
- 严格的内容保护和禁止扩写规则。

### 7.2 模型切换

DeepSeek 为主模型。以下情况触发有限次数重试或切换至通义千问：

- 请求超时。
- API 限流。
- 服务端错误。
- 响应格式无效。
- 连续自动校验失败。

两个模型均失败时，任务进入待重试队列，页面继续显示已有译文或英文原文。

模型适配层统一处理 API 地址、API Key、模型名、超时、重试和用量记录。
模型和 Prompt 均可在管理后台配置并版本化。

### 7.3 自动校验

校验至少包括：

- 所有保护占位符数量和值一致。
- 代码标识符、URL、数字和文件路径未被修改。
- HTML 或结构标记成对且顺序合法。
- 译文没有明显遗漏或异常扩写。
- 译文不包含模型解释、前言或 Markdown 围栏等额外内容。

校验不通过的译文不可进入当前发布版本。

## 8. 同步与更新

### 8.1 按需翻译

首次访问页面时：

1. 命中缓存则直接显示。
2. 未命中时先显示英文原文。
3. 页面进入高优先级抓取和翻译队列。
4. 翻译完成后提示中文版本已就绪。

### 8.2 后台逐步覆盖

未访问页面由低优先级任务逐步抓取和翻译。系统设置每日翻译额度和并发上限，
避免首次全站覆盖造成突发 API 费用。

### 8.3 每日变化检查

每天：

1. 检查 sitemap 页面清单。
2. 获取页面当前原文。
3. 比较页面和段落内容指纹。
4. 未变化内容不调用 AI。
5. 仅将新增或修改段落加入翻译队列。
6. 删除内容从当前版本移除，但保留必要的历史记录。

新版本必须在关键内容校验通过后原子发布，避免用户看到半更新页面。

## 9. 中英文统一搜索

搜索索引保存：

- 中文译文。
- 英文原文。
- 中文和英文标题。
- 页面 URL 与导航层级。
- 专业术语。
- API identifier 和代码标识符。
- 翻译状态和更新时间。

查询行为：

- 中文查询优先匹配中文译文，同时匹配英文术语。
- 英文查询优先匹配英文原文、术语和标识符。
- 混合查询拆分后同时检索。
- API identifier 支持精确匹配。
- 未翻译页面仍可通过英文原文检索。
- 搜索结果可切换中文或英文摘要。

PostgreSQL 负责英文全文检索和模糊匹配。中文内容在应用层分词后写入搜索字段，
减少对特定数据库扩展的部署依赖。

## 10. 管理后台

个人管理后台提供：

- DeepSeek 和通义千问连接配置。
- 模型、超时、重试和每日 Token 上限。
- System Prompt 与翻译 Prompt 版本管理。
- 专业术语保留列表。
- AI 初译编辑和人工校正。
- 英文原文差异查看。
- 抓取、翻译和失败任务查看。
- 每日 Token 消耗和模型切换记录。
- 最近同步、备份、磁盘和数据库状态。

API Key 不显示完整值，也不写入 Git。API Key 使用服务器端主密钥加密后存入
数据库；主密钥只通过服务器环境变量或 Docker secret 注入。

## 11. 登录与安全

### 11.1 单用户密码

- 密码本身不保存。
- PostgreSQL 仅保存使用 Argon2id 生成的带 salt 密码哈希。
- 首次密码通过服务器交互命令设置：

```bash
docker compose run --rm web admin set-password
```

- 命令要求输入两次，不回显密码，不将密码写入命令参数。
- 登录后可在管理后台验证旧密码并修改。
- 忘记密码时通过同一服务器命令重置。

### 11.2 会话

- 浏览器只保存随机会话 Cookie。
- Cookie 使用 `Secure`、`HttpOnly` 和合适的 `SameSite` 设置。
- 默认登录有效期为 30 天。
- 会话可在后台撤销。

### 11.3 公开入口保护

- 全站业务接口验证登录会话。
- 登录、搜索、抓取触发和翻译接口限制请求频率。
- 使用 HTTPS、安全响应头和请求体大小限制。
- 禁止公开注册。
- 设置 `robots.txt` 和 `noindex`，避免搜索引擎收录。
- 用于加密 AI API Key 的主密钥通过服务器环境变量或 Docker secret 注入。
- AI API Key 可在管理后台设置，但数据库中只保存经过认证加密的密文。

## 12. 数据模型

核心实体包括：

- `users`：单用户账号和密码哈希。
- `sessions`：登录会话。
- `pages`：规范化 URL、标题、层级和当前版本。
- `page_versions`：每次成功抓取的页面版本。
- `content_blocks`：结构化英文内容块及指纹。
- `translations`：译文、状态、模型、Prompt 和术语版本。
- `translation_edits`：人工校正历史。
- `glossary_terms`：保留术语和匹配规则。
- `prompt_versions`：提示词版本。
- `jobs`：抓取、翻译、索引和维护任务。
- `model_usage`：Token、费用估算、延迟和错误。
- `backup_runs`：备份和恢复验证记录。

所有外键删除规则必须防止误删人工校正和审计记录。

## 13. 备份与恢复

- 每天使用 PostgreSQL `pg_dump` custom format 生成压缩备份。
- 本机保留最近 14 天备份。
- 备份任务记录文件大小、校验和、开始时间和完成状态。
- 定期使用临时数据库执行 `pg_restore` 验证。
- 数据库备份可能包含加密后的 API Key 密文，但不包含解密所需的服务器主密钥。
- 第一版支持本机备份；部署时可增加 OSS、COS 或 OBS 异地副本。

只在同一服务器保存备份不能防范整机或磁盘损坏，因此异地副本作为推荐部署项。

## 14. 故障处理

- DeepSeek 失败：重试后切换通义千问。
- 两个模型都失败：保留英文或已有译文，任务延迟重试。
- Shopify 抓取失败：保留上次成功版本并显示同步时间。
- 页面解析不完整：不发布中文结构，显示英文和官方链接。
- 搜索索引更新失败：页面版本继续可读，索引任务单独重试。
- 数据库或磁盘异常：暂停新增翻译写入并在管理后台告警。
- 备份失败：保留旧备份并显示高优先级告警。

## 15. 部署与备案

- 云厂商和具体套餐在开发完成、部署前确定。
- 目标最低配置为 Linux、2 vCPU、4 GB RAM、约 80 GB SSD。
- 使用中国大陆节点和公开域名。
- 以湖北省个人主体办理 ICP 备案。
- 网站用途描述为个人技术文档阅读、翻译与检索工具。
- 不使用“Shopify 官方中文站”等可能造成官方关系误解的名称。
- 按适用要求完成公安联网备案并展示备案信息。

## 16. 测试策略

### 16.1 单元测试

- 页面规范化和分段。
- 内容指纹。
- 术语和占位符保护。
- 翻译结果校验。
- 模型故障切换。
- 搜索分词和排序。
- 密码哈希与会话过期。

### 16.2 集成测试

- Shopify 页面获取到数据库入库。
- 首次访问触发按需翻译。
- 变化段落单独重译。
- 人工校正优先级。
- DeepSeek 失败后切换通义千问。
- 中英文与 identifier 混合搜索。
- 备份与恢复。

### 16.3 端到端测试

- 登录、长效会话和退出。
- 阅读页面、中英文切换和滚动位置保持。
- 搜索结果进入对应中文或英文页面。
- 管理后台修改 Prompt、术语和译文。
- 模型不可用、抓取失败和旧版本降级展示。

## 17. 验收标准

- 未登录用户无法访问正文、搜索、管理或翻译接口。
- 代码块和代码注释与 Shopify 原文逐字一致。
- 锁定术语和技术标识符不会被翻译或修改。
- 已缓存页面可直接显示，不重复调用 AI。
- 每日更新只重译内容指纹发生变化的段落。
- DeepSeek 不可用时可以自动切换通义千问。
- 两个模型均失败时仍可阅读英文原文。
- 中文、英文和 API identifier 均能检索到对应页面。
- 人工校正可以保存，并在原文变化后准确标记待复核。
- 每日备份生成并保留 14 天，恢复验证可成功执行。
- 整个应用可在目标 2 vCPU、4 GB 服务器上通过 Docker Compose 运行。

## 18. 明确延后

以下内容不进入第一版：

- 多用户、角色和邀请系统。
- 移动应用。
- 浏览器扩展。
- 本地 GPU 模型。
- Elasticsearch 等独立搜索集群。
- 对 Shopify 高度交互工具的复刻。
- 无限制的全站即时预翻译。
